## Priority 6: Performance - JIT Coverage Expansion

Currently IRIS JIT achieves 0.17µs per op (native speed) for scalar arithmetic, but falls back to the tree-walker (100-500x slower) for anything else. Expanding JIT coverage is the single biggest performance lever.

### Tier 1: Loop/Collection JIT (biggest impact, unlocks benchmark parity with Python)
- [ ] `FoldPrim`: JIT-compile fold with scalar step functions (sum, product, max, min)
- [ ] `FoldLambda`: JIT-compile fold with inline lambda bodies
- [ ] `MapPrim`: JIT-compile map with scalar transforms
- [ ] `FilterPrim`: JIT-compile filter with scalar predicates
- [ ] `Call` / `Return`: JIT-compile function calls (enable multi-function JIT)

### Tier 2: Data Structure JIT (enables complex programs)
- [ ] `MakeTuple`: allocate and pack tuple on JIT stack
- [ ] `TupleGet`: extract tuple element by index
- [ ] `Concat` / `Take` / `Drop` / `Reverse` / `Zip`: list operations in JIT
- [ ] `LoadInputRef` index >= 6: extend from 6 to 16 arguments

### Tier 3: Graph Manipulation JIT (enables self-modifying programs at native speed)
- [ ] `SelfGraph`: return current program as Value::Program
- [ ] `GraphGetRoot` / `GraphGetKind` / `GraphGetPrimOp`: graph introspection
- [ ] `GraphSetPrimOp` / `GraphConnect` / `GraphDisconnect`: graph mutation
- [ ] `GraphAddNodeRt` / `GraphAddGuardRt` / `GraphAddRefRt`: node construction
- [ ] `GraphReplaceSubtree` / `GraphEval`: subtree operations
- [ ] `GraphNodes` / `GraphSetCost`: remaining graph ops

### Tier 4: Float64 Gaps
- [ ] `Mod (Float64)`: fmod
- [ ] `Pow (Float64)`: powf

### Expected Impact
| After completing | n-body | fannkuch | binary-trees | vs Python |
|-----------------|--------|----------|-------------|-----------|

## Priority 10: Community Training Pipeline

IRIS evolution produces optimized SemanticGraphs, not modified `.iris` source. Community contributions can improve the core language at three levels:

### Level 1: Program Submissions
- [ ] Accept `.iris` program submissions via package system (URL + BLAKE3 hash)
- [ ] Auto-extract GIN features from submitted programs
- [ ] Add to global training corpus for GIN-VAE retraining
- [ ] Track provenance: who submitted, when, which programs descended from it

### Level 2: Test Case Contributions
- [ ] Standardized spec format: `-- test: inputs -> output` comments
- [ ] Community spec repository (decentralized, Zig-style)
- [ ] Auto-evolve solutions for submitted specs, publish best results
- [ ] Difficulty scoring: how many generations to solve

### Level 3: Evolved Program Sharing
- [ ] Export evolved SemanticGraphs as shareable artifacts (JSON + BLAKE3 hash)
- [ ] Import evolved programs into local IrisRuntime
- [ ] Merge evolved embeddings from multiple sources into local GIN-VAE
- [ ] Federated training: each node trains locally, shares embedding updates

### Level 4: Corpus-Driven Language Improvement
- [ ] Periodic GIN-VAE retraining on expanded corpus
- [ ] Measure crossover quality before/after retraining (semantic similarity of offspring)
- [ ] Track which contributed programs most improved evolution outcomes
- [ ] Publish corpus statistics: total programs, diversity metrics, best solve rates
