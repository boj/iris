# IRIS Package Manifest Specification

## Overview

An IRIS project is defined by an `iris.toml` file at the project root. This manifest declares the project's name, version, entry point, dependencies, and build configuration.

## File Format

```toml
[package]
name = "my-project"
version = "0.1.0"
entry = "src/main.iris"
description = "A short description"

[dependencies]
# Path-based (local)
utils = { path = "../shared/utils" }

# Hash-based (content-addressed, immutable)
json = { hash = "a1b2c3d4e5f6..." }

# URL-based (downloaded + verified by hash)
http-client = { url = "https://iris-pkg.example.com/http-client-0.2.0.tar.gz", hash = "deadbeef..." }

[build]
features = ["syntax"]
optimize = true
```

## Resolution

1. **Path dependencies** resolve relative to the manifest file
2. **Hash dependencies** are looked up in `~/.iris/fragments/` (the persistent cache) or downloaded from a URL if provided
3. **URL dependencies** are downloaded once, verified by BLAKE3 hash, and cached

## Lockfile

`iris.lock` records the resolved BLAKE3 FragmentId for every dependency (including transitive ones). Generated by `iris lock` or automatically on first build.

```toml
[[fragment]]
name = "json"
hash = "a1b2c3d4e5f6..."
source = "hash"

[[fragment]]
name = "http-client"
hash = "deadbeef..."
source = "url:https://iris-pkg.example.com/http-client-0.2.0.tar.gz"
```

## CLI Commands

- `iris init` -- create a new `iris.toml` in the current directory
- `iris lock` -- resolve all dependencies and write `iris.lock`
- `iris build` -- compile the project (uses `iris.toml` entry point)
- `iris add <name> --hash <hash>` -- add a dependency
- `iris add <name> --path <path>` -- add a local dependency

## Design Principles

1. **Decentralized**: no central registry. Dependencies are URLs + BLAKE3 hashes (like Zig).
2. **Content-addressed**: the FragmentId IS the version. No semver needed for immutable content.
3. **Reproducible**: `iris.lock` pins every dependency by hash. Builds are deterministic.
4. **Incremental**: the fragment cache means repeated builds skip recompilation for unchanged dependencies.

## Future: Discovery

A lightweight index (JSON file hosted anywhere) maps human-readable names to URLs + hashes:

```json
{
  "packages": {
    "json": {
      "0.1.0": { "url": "...", "hash": "..." },
      "0.2.0": { "url": "...", "hash": "..." }
    }
  }
}
```

Multiple indexes can be combined (like Nix channels). No single point of control.
